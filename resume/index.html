소 개

처음, 직군 변경의 이유로는 서버 엔지니어 2년 동안 단순히 매뉴얼에 있는 명령어들을 타이핑만 한다는 느낌에 이직을 결심하게 되었고, 서버 직군을 하면서 Spring 기반의 웹 개발을 경험했지만 이미 정해진 패턴에 IF~ELSE만 붙이는 웹 개발은 미래가 없다고 생각이 들었습니다. 그래서 생각하는 개발을 하고자 휴식 기간 동안 C++를 배우면서 FTP를 만들었습니다. 처음에 아무것도 없는 빈 파일에서 첫 타이핑을 시작하기까지 매우 오래 걸렸던 기억이 있습니다. 어떻게 각 모듈을 분리해야 하는지 배운 적도, 해본 적도 없어서 실제 기능을 짜기 전부터 수차례 다시 구조를 만들기를 반복했었습니다. RFC에도 정의되지 않는 내용은 직접 패킷을 분석하면서 개발을 진행하였고 이런 탐구하는, 분석하는, 생각하는 개발이 적성이 맞는다고 더욱 확신했습니다.

그러던 중 C++ 개발을 하는 회사에 이직하였고, 첫 직장에서 퇴직을 한 지 1년 정도 흐른 후 지금의 대표님과 일을 하면서 이직 준비를 해도 된다는 대화를 하고 다시 첫 회사로 재입사하게 되었습니다.

계획대로 C++에 대해 더 공부하고자 했습니다. 이직 준비를 하면서 떠나도 좋다라는 말씀을 들었지만 맡은 업무 수행에서 기본은 하자는 생각에 Javascript를 배우게 되었습니다. 이전 웹 개발 경험에서 Javascript는 문자열 검사나 select 박스 정도 다루는 언어라고 생각했었습니다. 그러나 깊게 배워보니 C/JAVA와 비교하면 강제성은 없어 다루기는 까다롭다면 까다로웠지만, 그만큼의 자유를 허락하는 JS의 매력에 굉장히 빠져들었고, 현재 추세도 더는 브라우저에 의존하지 않는 점등의 이유로 JS, 프런트 엔드 개발자로 이직을 결심하게 되었습니다.

JS의 경험은 업무에서의 Jquery가 있었고, 개인적으로 했던 작업은 html5 canvas태그 기반의 차트입니다. 처음에는 d3.js와 같은 시각화 도구를 사용하려고 했지만 “단순히 써봤다.” 보다는 그것을 만들어보면 어떨까 생각하여 HTML5의 canvas 태그를 공부하였고, Chart.js 소스를 분석하여 초기화, 수치 계산 등의 힌트는 얻되 Copy/Paste는 하지 않으면서 실제 기능과 똑같이 구현하면서 JS를 익혔습니다. 그 후 단 하나의 JS파일을 나누어야 한다는 생각에 모듈 시스템, webpack을 적용/학습해나가면서 번역을 진행 중이며 React를 빠르게 학습 중입니다.

제가 생각하는 강점은 업무 대부분을 스스로 했던 것 같습니다. 일의 긴급성, 여러 방안 중에 어떤 선택이 최선인지에 대한 자문은 구했지만, 대부분 스스로 해결하고 있습니다. 그런 식으로 일을 해왔습니다. 무엇보다 개발이 재미있습니다.

또한, 작지만, 꾸준히 지속해서 무언가를 만들어 나가고 있습니다.
